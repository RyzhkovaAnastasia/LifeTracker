Life Tracker

Назначение 
Приложение для планирования деятельности в виде автоматизированного ежедневника с системой подкрепления. Приложение дает возможность создавать и выполнять: ежедневные дела, закреплять привычки, выполнять цели, а также получать награды, созданные пользователем, которые можно обменивать за внутри игровую валюту. 
Аналог проекта Habitica, ToDoist и прочие приложения для планирования и геймификации.

Функциональные требования
1.Пользователь может авторизоваться в приложении и получить доступ к доске с заданиями и вознаграждениями.
2.Незарегистрированный пользователь не имеет доступа к страницам приложения.
3.Зарегистрированный пользователь может:
a.Создавать/редактировать/удалять (далее – манипулировать) привычки
b.Манипулировать ежедневными заданиями
c.Манипулировать целями
d.Манипулировать вознаграждениями
e.Осуществлять поиск заданий и вознаграждений по тегам
f.Видеть статистику профиля по выполненным задачам по тегам и общим характеристикам
g.Повышать свой уровень и накапливать деньги за выполнение задач
h.Создавать соревнование между приглашенными людьми с фиксированным призом за победу
4.Пользователь может добавлять других пользователей к своим целям.

Требования к доске с заданиями

При входе в приложение после авторизации пользователь должен видеть матрицу с четырьмя списками: привычки, ежедневные задания, цели, вознаграждения. Привычки, ежедневные задания должны отображаться только те, что были заявлены без даты или на сегодняшний день. Цели и вознаграждения отображаются все. Выполненные задания не отображаются на матрице.
Сверху от матрицы должен располагаться мультиселект с выбором тегов. При выборе набора тегов должны отображаться только те задания и вознаграждения, которые содержат эти теги.
Так же должна быть кнопка для добавления заданий и вознаграждений. При их создании должно открываться модальное окно с настройками.

Требования к статистике
В статистике должны отображаться количество денег, уровень пользователя, общее количество выполненных заданий, купленных наград, количество выполненных заданий разделенные по уровням сложности, которые принадлежат определенным тегам с фильтрацией по датам.
Переход на уровень пользователем представляет собой достижение определенного количества очков опыта. Уровни будут представлены числами Фибоначчи, умноженные на 5 начиная со третьего значения (5 очков – достижение первого уровня).

Требования к привычкам 
Привычки должны содержать в себе название, описание, сложность, теги, периодичность, продолжительность выполнения (счетчик), дата начала, поощрение выполнения, наказание выполнения.
За выполнение привычки пользователь поощряется добавлением опыта и денег, а за выполнение действию обратного выработке привычки пользователь наказывается тем, что опыт и деньги отнимаются из счета.
Количество добавления и отнимания опыта зависит от сложности привычки. При длительном выполнении за привычку количество добавления опыта и денег уменьшается. При наказании продолжительность выполнения ставится равной нулю.
Формула по начислению денег: (уровень + сложность) / продолжительность выполнения 
Формула по взиманию денег: продолжительность выполнения / (уровень + сложность)
Формула по начислению опыта: сложность / 4
Формула по взиманию опыта: -1.5 * сложность

Требования к ежедневным заданиям
Ежедневные задания содержат название, описание, сложность, теги, периодичность, дату начала и конца, сложность, подзадачи, выполнение задачи.
Подзадачи содержат только название и являются вложенными в основную ежедневную задачу.
Если ежедневное задание выполнено, то оно исключается из списка задач.
Формула по начислению денег: сложность / 2
Формула по начислению опыта: сложность * 0.2

Требования к целям
Цель содержат название, описание, сложность, теги, дату конца, сложность, подзадачи, выполнение задачи.
Подзадачи содержат только название и являются вложенными в основную ежедневную задачу.
Если задание выполнено, то оно исключается из списка задач, выводится цель в список до тех пор, пока она не будет выполнена вне зависимости от даты.
Формула по начислению денег: сложность + (уровень / 10)
Формула по начислению опыта: сложность + (уровень / 10)


Требования к наградам
Награда содержат название, описание, цена. Награду можно получить только, если пользователю хватает на нее денег. Если награда куплена, она не удаляется из списка, ее можно купить еще раз. 

Технические требования
1.Ошибки должны логироваться
2.Серверная обработка исключений
